🔑 Normalization (3NF)

  • 1NF: All columns atomic, no repeating groups
  • 2NF: ROUTE extracted — no partial FK dependency in CONCESSION
  • 3NF: No transitive dependencies across any table
  • Route data stored once, referenced by N concessions
  • Admin data not repeated in DOCUMENT or APPROVAL

🔄 Transactions (ACID)

  • Signup: BEGIN → INSERT student → INSERT student_document × N → COMMIT
  • Apply: BEGIN → UPSERT route → INSERT concession → INSERT document → INSERT approval → COMMIT
  • Admin action: BEGIN → UPDATE concession → UPDATE approval → UPDATE document → COMMIT
  • Any step fails → ROLLBACK (Atomicity guaranteed)

🛡️ Constraints

  • PRIMARY KEY on all *_id columns
  • UNIQUE on enrolment_no, email, concession_id in approval
  • NOT NULL on all mandatory fields
  • CHECK on status, duration, year, expiry > issue
  • FOREIGN KEY with ON DELETE RESTRICT / CASCADE / SET NULL

📊 Indexes

  • idx_concession_student_id — fast student history lookup
  • idx_concession_status — fast pending/active filter
  • idx_concession_expiry — fast expiry-based queries
  • idx_document_concession_id — fast document lookup
  • idx_approval_concession_id — fast approval lookup
  • idx_student_document_student_id — fast registration doc lookup
